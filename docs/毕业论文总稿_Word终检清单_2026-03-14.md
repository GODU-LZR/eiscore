# 毕业论文总稿 Word 终检清单

整理日期：2026-03-14
适用对象：`docs/毕业论文总稿_目录版_2026-03-14.md` 合并到 Word 总稿后的最终排版检查
用途：作为提交前最后一轮人工核查清单使用

## 1. 结构检查

1. 设计总说明、英文设计总说明、第一章至第九章、致谢、参考文献的顺序是否一致。
2. `1.4 本文组织结构` 中的章节说明是否与实际目录一致。
3. 各章标题是否全部左对齐。
4. 是否仍存在“本章需补充”“建议插图”“后续再补”等编辑性文字。
5. 是否仍存在多余的空标题、单独存在的唯一子标题或断裂编号。

## 2. 字体与段落

1. 中文正文是否统一为 `宋体`。
2. 英文、数字是否统一为 `Times New Roman`。
3. 全文是否统一为 `两端对齐`。
4. 英文摘要和英文段落是否设置首行缩进。
5. 段前段后间距是否统一，没有靠手工空行撑版。

## 3. 页眉与页码

1. 封面是否无页码。
2. 目录是否无页码。
3. 设计总说明与英文部分是否使用罗马数字页码。
4. 正文是否使用阿拉伯数字页码。
5. 页眉是否按目录、设计总说明、正文分别设置。

## 4. 图表与公式

1. 图号是否统一为 `图X-X`。
2. 表号是否统一为 `表X-X`。
3. 图标题是否放在图下方。
4. 表标题是否放在表上方。
5. 正文中是否都出现了明确引用，如 `如图3-1所示`、`详见表5-2`。
6. 是否还存在“从下图可以看出”“如下表所示”但没有编号的写法。
7. 公式编号如存在，是否统一为 `（X-X）` 且右对齐。

## 5. 内容口径

1. 已实现内容是否只写：
   - 微前端基座
   - 人事管理
   - 物料管理
   - 应用中心
   - 移动端
   - 数据库中心架构
   - 轻量化语义增强
2. 流程能力是否统一写为：
   - 已完成基础配置与阶段性落地
3. AI 能力是否统一写为：
   - 原型探索
   - 阶段性验证
4. 销售、采购、生产、质量、设备、能耗等是否没有被写成“当前论文已完整实现的核心模块”。

## 6. 语言风格

1. 正文是否避免使用 `我`、`我们`。
2. 是否去掉了“老师要求”“论文写作角度”“后续定稿时”等元注释。
3. 是否避免口语化表达，如：
   - `可以看出`
   - `做成了什么`
   - `写成`
   - `后面再补`
4. 结论和展望是否区分清楚，没有把“未来计划”写成“已完成成果”。

## 7. 参考文献

1. 参考文献区是否只保留正式文献条目。
2. 是否删除“用途”“建议优先引用”“注意事项”等说明性文字。
3. 是否统一为学校要求的 `GB/T 7714-2015` 格式。
4. 正文引用编号是否与参考文献列表一一对应。

## 8. 打印前检查

1. 目录页码是否与正文页码一致。
2. 图题、表题是否居中且字号统一。
3. 空白页是否只保留打印规则所需页面。
4. PDF 导出后字体是否正常，没有乱码、错位、标题跳行异常。

## 9. 最后确认

提交前至少再做一次完整通读，重点检查：

1. 错别字
2. 病句
3. 图表引用漏写
4. 页码错位
5. 章节顺序错误
6. “已实现”和“原型验证”口径混写
