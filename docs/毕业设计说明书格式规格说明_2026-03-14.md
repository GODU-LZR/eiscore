# 毕业设计说明书格式规格说明

整理日期：2026-03-14
适用对象：广东海洋大学软件类毕业设计学生
适用文档：毕业设计说明书第一稿、第二稿、定稿 Word 文档
信息来源：毕业设计说明书格式规范培训会议记录、现有论文统稿材料
定位说明：本文档用于统一毕业设计说明书的版式、编号、图表、引用和打印检查口径，作为排版执行规范使用。

## 1. 编制目的

为避免毕业设计说明书在封面、页眉页脚、目录、图表编号、正文排版和打印环节出现反复返工，本文档对会议中明确的格式要求进行结构化整理，形成可直接执行的规格说明。

本文档重点解决以下问题：

1. 封面和基础信息填写不规范。
2. 正文章节结构与标题层级混乱。
3. 图表、公式和附录处理不符合要求。
4. Word 目录、页码和打印版排版不一致。
5. 全文字体、对齐方式、语言风格不统一。

## 2. 使用原则

本文档中的要求分为三类：

1. `必须`：属于提交前必须满足的强制要求。
2. `应`：属于默认执行要求，如有模板差异，以学院最终模板为准。
3. `建议`：属于优化版面和表达质量的推荐做法。

执行顺序应为：

1. 先完成正文内容。
2. 再统一处理标题样式、编号、图表和目录。
3. 最后处理页码、奇偶页和打印版检查。

## 3. 提交节点

1. 第一稿完整稿提交时间为 `2026-03-15`。
2. 第一稿提交时，内容和格式都应达到“可审阅”状态，不能只交正文半成品。
3. 后续修改可以继续优化表达，但基础格式错误不应留到第二稿再处理。

## 4. 封面与基础信息规格

### 4.1 封面字段填写

1. `届别`必须填写 `2026届`，不得填写入学年份。
2. `班级名称`必须填写全称，例如 `软件卓越1221`，不得写简称。
3. `中文题目`必须与教务系统登记题目完全一致，不得自行增删字词。
4. `英文题目`必须与中文题目语义一致，并与教务系统或最终登记版本保持一致。
5. `指导教师`、`副指导教师`、`职称`等信息必须按导师提供名单填写，不得凭记忆填写。
6. `答辩时间`在最终通知未明确前，可按会议建议暂填 `5月底` 或 `8月初`，定稿前再统一修正。

### 4.2 封面字体与表格

1. 中文标题必须使用 `方正姚体`。
2. 如本机未安装 `方正姚体`，应先补装字体后再排版。
3. 封面中的表格内容应使用 `仿宋_GB2312` 小三号。
4. 打印前应确认打印环境支持相关字体；如存在字体缺失风险，应优先导出为 PDF 再打印。

## 5. 页眉、页脚与页码规格

### 5.1 页眉

1. 页眉内容必须与当前部分一致，不得全文沿用同一页眉。
2. 目录部分应使用目录对应页眉。
3. 设计总说明和英文部分应使用各自对应页眉。
4. 旧模板中若存在 `Abstract` 页眉，应按会议要求统一改为 `Introduction`。

### 5.2 页码

1. 封面不得显示页码。
2. 目录不得显示页码。
3. 设计总说明及其英文对应部分应使用罗马数字页码。
4. 正文部分应使用阿拉伯数字页码。
5. 目录、摘要、正文在打印版中均应从奇数页开始。
6. 如上一部分结束于偶数页，打印版应补空白页；电子版可不补空白页。

## 6. 正文结构与标题层级规格

### 6.1 章节逻辑

1. 章节安排必须符合软件工程说明书逻辑。
2. `可行性分析`如保留，应放在前部，不得放在结尾。
3. 标题应准确概括内容，不得出现明显口语化、过程化或不规范表述。
4. 不得使用诸如“开发工具版本号”“某时间段收益流水”之类不适合作为论文章节标题的表达。

### 6.2 标题对齐

1. 各级标题必须顶格左对齐。
2. 各级标题不得居中。
3. 各级标题不得手动缩进。

### 6.3 标题层级冗余处理

1. 若某一级标题下只有一个子标题，应删除该冗余子标题。
2. 不得出现“有二级标题但该二级标题下只有唯一一个三级标题”的空转结构。

## 7. 编号体系规格

### 7.1 正文层级编号

1. 正文层级编号应采用 `（一）`、`1）`、`（a）` 等带括号形式。
2. 不得在正文层级中使用容易与章节号混淆的裸编号形式，如 `一、`。
3. 同一层级编号格式必须全文统一，不得混用。

### 7.2 段内列举编号

1. 段落内部的并列事项，建议使用带圈数字，如 `①②③`。
2. 同一段内的编号样式应统一，不得在同一层中混用中文序号、阿拉伯序号和短横线。

## 8. 段落、字体与语言风格规格

### 8.1 段落排版

1. 全文应设置为 `两端对齐`。
2. 英文段落应设置首行缩进 `2字符`。
3. 新章开始不强制分页，但应与前文空两行。
4. 同一章内二级标题之间的空行规则必须统一：要么统一空一行，要么统一不空行。

### 8.2 字体规范

1. 中文正文统一使用 `宋体`。
2. 英文和数字统一使用 `Times New Roman`。
3. 图题、表题字号建议使用 `小五号`。

### 8.3 语言风格

1. 正文不得使用 `我`、`我们` 等第一人称。
2. 应统一使用 `本设计`、`本课题`、`本文` 等客观表述。
3. 不得使用明显口语化或答辩式表达，如“可以看到”“大家都知道”“下面这个图就是”。

### 8.4 英文细节

1. `Keywords` 后应使用英文冒号。
2. 英文分号、逗号、冒号后应保留英文空格。
3. 中英文标题的大小写应符合英文标题规范：名词、形容词、副词、感叹词首字母大写，冠词、介词、连词通常小写。

## 9. 图表规格

### 9.1 基本要求

1. 所有图、表必须同时具备 `序号` 和 `标题`。
2. 图与表必须分开编号。
3. 图和表的编号必须采用二级编号格式，例如 `图3-5`、`表2-1`。
4. 不得使用三级编号形式。

### 9.2 图表标题位置

1. 图标题应置于图下方。
2. 表标题应置于表上方。
3. 图题和表题建议居中排版。
4. 图题和表题字号建议为 `小五号`。

### 9.3 图表与正文关系

1. 正文中必须先说明图表用途，再插入图表。
2. 正文中必须明确引用图表编号。
3. 引用表述应写为 `如图3-5所示`、`详见表2-1` 等规范形式。
4. 不得写成 `从下图可以看出`、`如下表所示` 这类无编号引用。
5. 不得为了凑页数机械插图，图表必须服务于正文论述。

### 9.4 图表间距

1. 图题与后续正文之间应保留适当段后间距。
2. 表题与表格之间应保留适当前后间距。
3. 若图表与上下文过于贴近，应通过段前段后间距进行微调，不建议靠手动回车堆空白。

## 10. 公式规格

1. 公式编号应采用 `（3-10）` 形式。
2. 编号应放置于公式右侧。
3. 公式与编号之间应保持空白，不得加入虚线、点线或其他装饰符号。
4. 正文引用公式时，应明确写出对应编号。

## 11. 目录生成规格

1. 目录必须使用 Word 自动生成功能生成。
2. 不得手工输入目录项和页码。
3. 每次修改标题、页码或分页后，必须重新更新目录。
4. 提交前必须核对目录标题与正文标题是否完全一致。
5. 提交前必须核对目录页码与正文实际页码是否一致。

## 12. 附录与代码处理规格

1. 大篇幅代码不得出现在正文中。
2. 代码、长 SQL、长脚本、长配置清单应放入附录。
3. 如附录无内容，应删除附录页，不得保留空白附录。
4. 正文中如确需引用代码，只保留少量关键片段，并以设计说明为主，不得变成代码展示文档。

## 13. 打印版与电子版处理规格

### 13.1 电子版

1. 电子版按自然顺序排版即可。
2. 电子版无需为奇数页打印规则额外插入空白页。

### 13.2 打印版

1. 目录、摘要、正文均应从奇数页开始。
2. 如需双面打印，应检查奇偶页顺序是否正确。
3. 打印前必须确认 Word 版与 PDF 版页码一致。
4. 打印前必须确认目录页码与正文页码一致。

## 14. 提交前终检清单

提交前至少完成以下检查：

1. 封面届别是否为 `2026届`。
2. 班级名称是否使用全称。
3. 中英文题目是否与教务系统完全一致。
4. 导师信息、职称、答辩时间是否准确。
5. 中文是否统一为宋体，英文数字是否统一为 `Times New Roman`。
6. 全文是否统一为两端对齐。
7. 标题是否全部左对齐且层级一致。
8. 目录是否为自动生成，且已更新。
9. 图、表、公式是否完成编号。
10. 图表是否都在正文中进行了说明和引用。
11. 是否仍存在大段代码直接放在正文中的情况。
12. 页眉是否按章节部分切换。
13. 设计总说明和正文页码格式是否区分。
14. 打印版是否满足奇数页起始要求。
15. 全文是否通读并检查错别字、病句和逻辑不通顺之处。

## 15. 与现有文档的配合方式

本文档只解决“格式怎么排”的问题，不替代以下材料：

1. `docs/THESIS_DRAFT_REQUIREMENTS_SUMMARY_2026-03-13.md`
   用于说明“说明书应该写什么”。
2. `docs/论文初稿统稿清单_2026-03-14.md`
   用于说明“总稿如何合并与补齐”。
3. `docs/论文截图补图规范与插入顺序_2026-03-14.md`
   用于说明“截图如何整理与插入”。
4. `docs/国标引用格式核查与修订建议_2026-03-14.md`
   用于说明“参考文献如何改成最终国标格式”。

## 16. 执行建议

如果现在要赶第一稿，建议按以下顺序处理：

1. 先统一封面、标题样式、正文字体和段落格式。
2. 再统一图号、表号、公式号和正文引用语句。
3. 然后更新目录、检查页码和页眉。
4. 最后分别导出电子版和打印版，完成终检。

一句话原则：先把内容写完整，再把格式一次性收口，不要在初稿阶段反复做零碎排版。
