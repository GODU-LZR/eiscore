# 毕业论文初稿目录与章节提纲

题目：面向湛江中小制造企业的企业信息化系统核心设计与实现——以广东南派食品有限公司为例

适用说明：

1. 本文档用于毕业论文初稿撰写。
2. 内容结合当前项目代码、开题报告和中期报告整理。
3. 写作时可先按本提纲补全正文，再统一插入图、表、参考文献和格式。

## 一、设计总说明

设计总说明建议分为三段：

1. 第一段：介绍课题背景与研究意义。
2. 第二段：介绍 EISCore 系统的主要内容、技术路线和系统特点。
3. 第三段：概述全文结构，说明每一章写了什么。

补充要求：

1. 中文约 1500 字。
2. 需同步准备英文版。
3. 这一部分不要写得太散，要像整篇论文的总览。

## 二、推荐目录结构

### 摘要

写作重点：

1. 简述研究背景。
2. 说明本文设计并实现了什么系统。
3. 概括主要技术路线，如 PostgreSQL、PostgREST、Vue3 微前端、BPMN、本体语义增强等。
4. 说明系统目前完成的主要功能与阶段性成果。
5. 最后点出研究意义。

关键词建议：

1. 企业信息化系统
2. 中小制造企业
3. 数据库中心架构
4. 微前端
5. 本体语义
6. 工作流

### Abstract

要求与中文摘要对应，术语尽量统一。

### 第一章 绪论

建议小节：

#### 1.1 研究背景

可写内容：

1. 中小制造企业数字化转型的现实需求。
2. 湛江地区制造企业，特别是农海产品加工企业的信息化痛点。
3. 传统 ERP 和轻量 SaaS 在成本、定制化和运维上的局限。

#### 1.2 研究目的与意义

可写内容：

1. 说明为什么要设计 EISCore。
2. 从理论层面说明数据库中心架构、低代码、AI 辅助和本体语义建模的研究价值。
3. 从应用层面说明对南派食品此类企业的实际意义。

#### 1.3 国内外研究现状

可写内容：

1. 国外在 Composable Enterprise、Micro-frontends、PostgREST、BPMN、Generative UI 等方面的发展。
2. 国内在制造业信息化、低代码平台、定制化开发等方面的现状。
3. 分析现有方案的优点和不足，引出本文的研究切入点。

#### 1.4 本文主要研究内容

可写内容：

1. MMS 核心模块设计与实现。
2. HMS 统一权限与身份治理。
3. 应用中心和动态应用配置。
4. BPMN 流程引擎集成。
5. 轻量本体语义增强机制设计与应用。

#### 1.5 论文结构安排

按章节简要说明全文内容。

#### 1.6 本文创新点

建议写 3 点左右，不宜写太大：

1. 采用数据库中心架构，弱化传统后端服务层。
2. 将本体语义增强机制引入企业信息化系统，用于统一表、字段、关系和流程语义。
3. 将动态应用配置、流程设计和语义治理整合到同一底座中。

## 四、初稿撰写顺序建议

为了赶第一稿，建议按下面顺序写：

1. 先写摘要、第一章、第二章。
2. 再写第四章需求分析。
3. 接着写第五章概要设计和第六章数据库设计。
4. 然后补第七章详细设计和第八章系统实现。
5. 最后写第九章测试、第十章总结与展望、致谢。

## 五、你这篇论文的写作提醒

1. 论文主线不要写散，要始终围绕“面向中小制造企业的信息化底座系统设计与实现”展开。
2. 本体论不要写得太玄，建议统一写成“轻量本体语义增强机制”。
3. AI 开发器要写成“原型验证与部分链路实现”，不要写成已经完全成熟。
4. 数据库中心架构是你的核心特色之一，前后章节都要围绕它展开。
5. 论文正文不要直接贴大量代码，重点是设计逻辑、实现过程和结果展示。
