# 长报告核验与可采纳结论

整理日期：2026-03-15
对应材料：用户提供的《2026年中文学术论文AIGC检测深度研究与智能体提示词重构报告》

## 1. 总体判断

这份长报告不是完全不能用，但它把三类内容混在了一起：

1. `可核实的公开事实`
2. `可以转化为写作改进原则的经验判断`
3. `大量未经证实、表述过强、甚至明显可疑的技术推断`

因此，它不能被当作论文中的事实依据，也不适合直接照搬成“降检测提示词”。
更稳妥的用法，是把其中少数可核实部分当作背景，把少数合规写作原则转成内部改稿标准。

### 2.2 Turnitin AI Writing Report

已核到公开页面：

2. 官方明确承认该模型可能误判，不能单独作为处分依据。
3. 低比例区间存在更高误报风险。

### 2.3 DetectGPT

<https://arxiv.org/abs/2301.11305>

可保留的结论：

1. DetectGPT 确实是公开的零样本文本检测方法。
2. 它基于概率曲率，不依赖单独训练分类器。

### 2.4 CodeBERT

已核到论文：

<https://arxiv.org/abs/2002.08155>

可保留的结论：

1. CodeBERT 确实是代码与自然语言联合预训练模型。

### 2.5 BrIdGe

2. 它面向的是 `localization / 翻译与本地化重写`，不是专门针对学术 AIGC 检测规避提出的方法。

## 3. 可借鉴但不能直接照搬的部分

这份长报告里有一些思路可以转成合规改稿原则，但不能按原文那种“对抗检测”表述使用。

### 3.1 可吸收的原则

1. 不要让整段都保持同一种句长、同一种句式。
2. 技术段落不要只写“技术是什么、优点是什么”，而要落到项目职责和工程取舍。
3. 双语摘要不要机械逐句对译，英文版应按英文写作习惯重新组织。
4. 优先补真实模块、真实页面、真实表名、真实部署链路，而不是只做同义词替换。

### 3.2 转化后的合规表达

不要写：

1. “绕过检测”
2. “对抗算法”
3. “规避模型识别”

应改写成：

1. “降低模板化表达”
2. “减少空泛和机械重复”
3. “增强项目事实锚点”
4. “提升双语摘要的自然写作差异”

## 4. 明显不宜采纳或无法证实的部分

以下内容不能直接作为事实依据使用，也不应进入论文或内部正式说明：

### 4.1 “2026年CAC强制要求平台集成多层级AIGC检测算法”

目前未核到与你这段话等价、可直接支持这种强表述的公开权威依据。
这类说法过强，不建议使用。

### 4.2 “ShanghaiTect 检测器”

未核到其作为中文学术论文 AIGC 检测系统的可靠公开来源。
相反，公开检索里 `ShanghaiTech / ShanghaiTect` 更常见的是 `人群计数数据集` 相关内容，而不是论文 AIGC 检测平台。

因此：

1. 不能把 `ShanghaiTect` 当成已知权威学术论文检测系统来引用。
2. 也不能把“ShanghaiTect 对 EISCore 打分 62.2/100”这类数字当成可信分析结果。

### 4.3 “CodeBERT 直接用于高校学术论文检测并形成聚类中心判定”

虽然 CodeBERT 本身真实存在，但“高校检测平台具体如何把 CodeBERT 嵌入论文检测流程，并据此判定你的论文”这一点，在长报告中没有给出可核实来源。
这类推断可以作为个人技术猜测，不能当作事实。

### 4.4 “对 EISCore 初稿的精确困惑度、突发性、嵌入异常度评分”

这些数字未见到对应的真实实验流程、模型参数、样本分段和复现实验。
因此不能当成可靠测量结果。

### 4.5 “通过提示词强制制造认知摩擦、低概率词、情绪爆点”

这类写法风险很高：

1. 容易把论文改得不自然。
2. 容易损害学术表达质量。
3. 容易引入事实失真和过度戏剧化描述。

不建议按这类指令改论文。

## 5. 对当前 EISCore 论文最有价值的实际建议

结合本次核验，最值得保留的不是那套“对抗检测话术”，而是下面这些更稳的做法：

1. 绪论少写宏观空话，多落到南派食品的具体问题。
2. 技术介绍少写教科书定义，多写 `Vue 3 / qiankun / PostgREST / RLS / BPMN / 轻量语义 / Agent Runtime` 在项目里各自承担什么职责。
3. 实现章节少写“系统已实现……”，多写页面职责、运行链路、表结构和函数。
4. 英文摘要不要逐句硬译中文摘要，而要按英文论文常见逻辑重新组织。
5. 所有新增内容都要以仓库中的真实页面、接口、表或脚本作为锚点。

## 6. 结论

一句话概括：

`这份长报告里，公开规则和少量写作原则可以吸收；大量关于“2026检测系统”“具体算法命中方式”“精准打分”的说法不宜当真，更不能直接作为论文或改稿依据。`

对当前项目最稳的做法仍然是：

1. 用真实实现锚点改论文
2. 用合规方式降低模板感
3. 把注意力放在截图、图表、参考文献和格式终检上
