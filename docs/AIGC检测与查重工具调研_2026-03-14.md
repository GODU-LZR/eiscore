# AIGC 检测与查重工具调研

整理日期：2026-03-14
用途：用于说明当前 AIGC 检测工具的可靠性、是否存在所谓“提示词降到 15%”的稳定方法，以及目前可用的开源或低门槛自测工具。

## 一、结论先说

### 1. 关于“把 AIGC 率稳定降到 15%”

联网检索后的结论是：**没有可靠证据表明，存在一种稳定、可复现、对学校系统普遍有效的“提示词方案”，可以把 AIGC 检测率稳定降到某个固定百分比，例如 15%**。

原因主要有三点：

1. 不同学校使用的检测工具不同，算法、语料、阈值和版本都可能不同。
2. 同一种工具在不同时期也会更新。例如 Turnitin 官方在 **2026 年 2 月 16 日**更新了 AI writing detection model，并明确提到开始增强对 AI bypasser tools 的识别能力。
3. AIGC 检测本身并不稳定。很多高校和研究文献都指出，这类检测容易出现误报和漏报，不能被视为绝对证据。

所以，网上那种“用某个提示词把 AIGC 率压到 15%”的说法，更接近经验帖或营销话术，不适合作为你论文交稿前的可靠策略。

### 2. 更稳妥的做法

如果你的目标是**降低误判风险**，而不是去“规避检测”，更稳妥的方式是：

1. 让正文真正回到你自己的写作逻辑。
2. 加入与你项目强绑定的真实细节。
3. 保留你自己的改稿痕迹、文献检索痕迹和图表制作痕迹。
4. 把参考文献、图表、测试数据、项目结构与正文一一对应。

这类做法不能保证任何工具给出某个固定百分比，但能显著降低“模板味”“空话味”“AI 套话味”导致的风险。

### 3. 关于开源检测工具

有开源工具可以做**参考性自测**，但要明确：

1. **没有开源工具能够准确复现学校的 AIGC 检测结果。**
2. **没有开源工具能够准确复现学校的查重系统结果。**
3. 开源工具更适合做“趋势判断”，不适合把输出百分比当成最终依据。

## 二、联网核到的关键事实

### 1. Turnitin 官方仍在更新 AI 检测模型

Turnitin 官方指南页面显示，其 AI writing detection model 在 **2026 年 2 月 16 日**有更新，并新增了对 **AI bypasser tools** 的识别说明。这个信息非常重要，因为它说明：

1. 检测模型是持续变化的；
2. 即使某些“人类化提示词”曾经在旧模型上有效，也不代表现在还有效；
3. 试图依赖所谓“降 AIGC 提示词”本身就存在失效风险。

### 2. 多所高校公开表示 AI 检测并不可靠

Vanderbilt University 的官方支持页面公开说明，经过测试后，他们决定停用 Turnitin 的 AI detector，并明确表示无法确认该工具是否足够可靠。其 generative AI 学术诚信页面也指出，学校**不提供机构级支持的 AI 检测工具**，并提醒第三方检测工具存在隐私和可靠性问题。

这说明一个现实情况：**高校内部也并没有把 AIGC 检测结果当作绝对证据。**

### 3. 学术研究普遍认为“可靠检测 AI 文本”很难

我检索到的论文和综述多数支持同一方向：

1. `Can AI-Generated Text be Reliably Detected?` 认为现有 AI 文本检测面对攻击和改写时并不稳健。
2. `Testing of detection tools for AI-generated text` 也得出类似结论，即现有检测工具在真实场景下准确性和可靠性都有限。

因此，**检测结果更像一个风险信号，而不是司法级别证据**。

## 三、如果不搞“绕过”，怎样合规降低风险

下面这些方法属于正常学术写作优化，不是规避检测。

### 1. 把“通用套话”改成“项目细节”

比起反复写“随着数字化转型不断发展”“具有重要意义”这类泛化表达，更应该多写：

1. 南派食品的业务痛点；
2. 你为什么引入轻量化语义模块；
3. 库存批次、入库、出库、流程状态映射的具体设计；
4. 你项目当前哪些模块完成了，哪些只是原型验证。

越具体、越贴项目，越不像模板文本。

### 2. 保留“你自己的判断句”

例如：

1. 为什么你把质量、生产、采购不再写成当前独立完成模块；
2. 为什么你把轻量化语义模块单独提出来；
3. 为什么你采用数据库中心架构而不是传统后端分层。

这些“判断句”和“取舍说明”非常能体现你的作者痕迹。

### 3. 不要机械改写，要重新组织

如果只是把 AI 生成文本做同义词替换，风险并不低。更稳的方式是：

1. 先确定你要表达的要点；
2. 按你自己的顺序重写；
3. 把段落结构改掉；
4. 把真实项目细节补进去。

### 4. 图、表、文要互相对应

老师其实很看重这个：

1. 第三章的需求图要和文字描述对应；
2. 第四章的架构图要和当前实际实现一致；
3. 第五章的表结构要和数据库脚本一致；
4. 第七章的截图要和实现内容一致；
5. 第八章的测试表要和测试报告一致。

如果这些东西能一一对应，论文整体可信度会明显提高。

### 5. 保留写作过程证据

建议你保留：

1. 开题报告、中期报告、初稿修改记录；
2. 参考文献检索记录；
3. 图表源文件；
4. SQL 脚本、前端页面、测试报告；
5. 每一版 Word 的修改历史。

如果真的被问到“是否大量依赖 AI”，这些过程证据比任何单一检测值都更有说服力。

## 四、有没有开源工具可以检测 AIGC 率

有，但请把它们当作**参考工具**。

### 1. 可参考的开源/开放工具

#### GLTR 系路线索

GLTR 是比较经典的 AI 文本可视化分析方向，GitHub 上有相关实现或衍生项目。
适合做：观察文本是否呈现高概率 token 分布特征。
不适合做：给出学校可接受的“最终 AIGC 百分比”。

#### `promptslab/openai-detector`

GitHub 上可见的开源分类器实现。
适合做：本地快速试验。
不适合做：当成正式学术检测结果。

#### `YuchuanTian/AIGC_text_detector`

公开论文对应代码，偏研究型。
适合做：研究性对比、自己跑实验。
不适合做：直接拿来判断你的论文是否一定安全。

#### `gpt-who`、`MAGE`、`DetectAnyLLM`

这些更多是论文公开代码或研究项目。
优点：透明，可自己跑。
缺点：部署成本较高，且不同模型、不同文本域下波动很大。

#### Manuscript

有“开源 detector”定位，但更偏通用内容检测平台。
是否适合中文毕业论文场景，还要自己验证。

### 2. 我对你的实际建议

如果你只是为了论文交稿前做自测，不建议你花很多时间自己部署一堆开源检测器。更实用的方法是：

1. 选 1 到 2 个开源 AI 文本检测器做交叉参考；
2. 不看它们的百分比，主要看它们是否一致地把某些段落判成高风险；
3. 如果多个工具都指向同一段，就重点回去改那一段；
4. 修改后再做一次对比。

也就是说，**它们适合定位高风险段落，不适合拿来赌最终检测分数。**

## 五、有没有开源工具可以检测“查重率”

严格来说，**没有开源工具能准确复现学校的查重率**。原因是学校常用系统（如 Turnitin、知网、维普、万方等）的核心能力并不只是算法，而是：

1. 拥有庞大的付费数据库和论文库；
2. 有专有的分句、比对和排除规则；
3. 学校通常还有自己的参数设置。

因此，开源工具最多只能做“近似相似度检查”，不能等同于学校最终查重。

### 1. 可参考的开源或低门槛文本查重工具

#### WCopyFind

比较老牌的文本相似性比较工具。
适合做：本地文档间相似内容比对。
不适合做：和知网、Turnitin 一对一对标。

#### UnPlag

可用于文本和代码的相似性检查。
适合做：小范围文本重复排查。
不适合做：正式论文最终查重替代。

#### 其它 GitHub 上的 plagiarism-detection 项目

很多项目是针对代码、短文本或局部文本匹配，真正适合中文毕业论文整篇查重的并不多。

### 2. 更现实的查重预检办法

如果你想在正式学校查重前降低风险，建议优先这样做：

1. 用本地检索查自己论文里是否大段重复开题报告、中期报告；
2. 检查是否整段照搬文献综述或技术定义；
3. 检查是否多个章节反复出现近乎相同的段落；
4. 特别检查“研究背景”“技术介绍”“总结”这三类最容易模板化的部分。

## 六、对你这篇论文最实用的处理建议

结合你当前这份论文，我建议按这个顺序处理：

1. 先统一所有章节口径，确保“已完成”和“原型验证”不乱。
2. 补第七章系统截图，把真实页面和正文一一对应。
3. 统一参考文献为最终 `GB/T 7714-2015` 格式。
4. 再用 1 到 2 个开源 AIGC 工具做“高风险段落扫描”。
5. 对被多次判高风险的段落，按你的项目真实细节重新写。
6. 不追求某个固定“AIGC 百分比”，而追求整篇论文的真实性、一致性和可解释性。

## 七、我建议你不要相信的说法

以下说法目前都不建议直接相信：

1. “某个提示词能稳定把 AIGC 压到 15%。”
2. “某个人类化工具能对所有学校系统都有效。”
3. “某个免费网站测出来 8%，学校也一定差不多。”
4. “把句子改短一点、插点口语就一定安全。”

这些说法最大的问题不是“完全没用”，而是**不可验证、不可迁移、不可复现**。

## 九、一句话结论

对你现在最有用的不是“找一个提示词把检测压到 15%”，而是把论文改成**真实项目细节充分、图表证据完整、文献来源可信、写作逻辑属于你自己**的版本。只要这四点扎实，风险会比盯着某个网页百分比更可控。
