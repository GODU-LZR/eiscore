## 3. 当前判断

从这一轮结果看，继续整篇按统一风格重写，并没有稳定把风险压下来。相反，以下几类段落会被启发式模型持续拉高：

1. 设计总说明里的背景总述。
2. 轻量化语义与 Agent Runtime 联动说明。
3. 需求分析中的销售订单状态描述。
4. 数据库设计里连续出现的“定义式表结构说明”。
5. 第七章和第九章中的模块总结句、整体完成情况总结句。

也就是说，问题已经不是“内容不够真实”，而是这些段落一旦被统一改成较整齐的书面风格，就会再次出现过强的模板感。

## 4. 当前推荐

如果目标是继续写论文而不是反复洗正文，当前更稳的底稿仍然是：

- [毕业论文最终主稿_终版_补闪念语义冷库模式_润色版_2026-03-15.docx](/home/lzr/eiscore/docs/毕业论文最终主稿_终版_补闪念语义冷库模式_润色版_2026-03-15.docx)

原因是：

1. 这版已经补齐了闪念应用、Agent Runtime、轻量化语义联动、数据表格应用联动和冷库模式。
2. 这版在此前本地启发式检测中表现仍然是更稳的。
3. 后续更适合在这版上补图、调格式和做极少量人工修订，而不是继续整篇重写。
