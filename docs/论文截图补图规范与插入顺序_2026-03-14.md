# 论文截图补图规范与插入顺序

题目：面向湛江中小制造企业的企业信息化系统核心设计与实现——以广东南派食品有限公司为例
版本：截图整理版
整理日期：2026-03-14
用途：用于统一系统截图的命名方式、补图顺序和论文插图位置，避免后期 Word 排版混乱。

## 一、截图文件命名规范

建议所有截图统一放在一个单独目录中，并按照“图号 + 简短中文名”的形式命名。

推荐命名规则：

`图号_页面名称.png`

例如：

- `图7-1_系统基座首页.png`
- `图7-3_员工档案管理页面.png`
- `图7-11_入库页面.png`
- `图7-19_本体关系工作台总览.png`

如果你希望进一步避免文件混乱，也可以采用下面这套更稳的格式：

`chapter7/图7-1_系统基座首页.png`

## 二、截图整理目录建议

建议你在 `docs` 下面单独建一个截图目录，例如：

`/home/lzr/eiscore/docs/thesis-images`

目录建议分成两层：

1. `chapter3`：第三章相关图片
2. `chapter4`：第四章相关图片
3. `chapter5`：第五章相关图片
4. `chapter7`：第七章系统实现截图

当前最需要补的是 `chapter7`。

## 三、截图统一要求

为了让论文看起来更规范，建议所有截图尽量满足以下要求：

1. 统一使用浅色界面，不要混入深色模式截图。
2. 尽量裁掉浏览器多余工具栏，只保留页面主体和少量地址栏信息。
3. 同一章的截图宽度尽量一致。
4. 页面中如果有测试数据，尽量避免明显无意义的数据或过于杂乱的内容。
5. 移动端截图建议统一为竖屏比例，桌面端截图统一为横屏比例。

## 六、插图题注建议写法

后续在 Word 中插图时，图题建议统一采用下面格式：

- 图7-1 系统基座首页
- 图7-3 员工档案管理页面
- 图7-11 入库页面
- 图7-19 本体关系工作台总览

不要写成：

- 系统截图1
- 页面展示
- 功能实现图

题注越具体越好，这样老师一眼就能看出页面是什么。

## 七、排版顺序建议

补图时建议按这个顺序进行：

1. 先补第七章高优先级截图
2. 再把第三章到第五章已有 SVG 图插入 Word
3. 最后统一调整所有图题和正文中的“见图X-X”

## 八、当前最实用的执行方式

你接下来最省事的做法是：

1. 先新建 `docs/thesis-images/chapter7`
2. 按本清单文件名逐张截图
3. 截完直接插入第七章对应位置
4. 最后统一调整图片大小和题注

这样做比后面一边找图一边改图题效率高很多。
