## 二、用例规约表

### 表3-3 用例规约：销售订单下推与计划生成

| 项目 | 内容 |
| --- | --- |
| 用例名称 | 销售订单下推与计划生成 |
| 参与角色 | 销售文员、PMC计划员 |
| 前置条件 | 已维护客户档案、客户物料清单和销售合同；系统中已存在可用基础物料和库存数据 |
| 触发条件 | 销售文员根据客户合同创建销售订单，并提交至计划环节 |
| 基本事件流 | 1. 销售文员录入或选择销售合同。2. 系统根据合同信息下推生成销售订单。3. PMC计划员查询成品库存与订单数量。4. 若库存满足，则标记为可直接发货。5. 若库存不足，则生成生产计划并进入缺料分析。6. 系统记录订单状态，供销售、PMC和仓库共同查看。 |
| 异常事件流 | 1. 客户物料映射不存在时，系统提示补充客户物料清单。2. 销售订单关键信息缺失时，不允许提交到计划环节。3. 库存数据异常或无法读取时，PMC只能先保留订单，待人工确认。 |
| 后置条件 | 销售订单与后续计划状态建立关联；订单进入“可发货”或“待排产”状态。 |

### 表3-4 用例规约：采购入库与来料检验

| 项目 | 内容 |
| --- | --- |
| 用例名称 | 采购入库与来料检验 |
| 参与角色 | 采购人员、质检人员、仓管员 |
| 前置条件 | 已存在采购订单；物料已到厂；质检项目或检验附件规则已预先配置 |
| 触发条件 | 采购人员或仓库登记到货，发起来料检验和入库处理 |
| 基本事件流 | 1. 仓管员登记到货信息。2. 系统关联对应采购订单。3. 质检人员创建来料检验单。4. 检验合格后，仓管员办理采购入库。5. 系统生成批次记录并写入库存流水。6. 入库结果可供后续领料、追溯和库存台账查询使用。 |
| 异常事件流 | 1. 来料检验不合格，则不得完成正式入库。2. 检验结果暂未出具时，系统保留“待补录”状态，并允许后续补传附件或补录结论。3. 采购订单不存在或物料不匹配时，系统提示到货登记失败。 |
| 后置条件 | 采购单、检验单、入库单和库存批次建立关联；库存台账同步更新。 |

### 表3-5 用例规约：生产领料与补料

| 项目 | 内容 |
| --- | --- |
| 用例名称 | 生产领料与补料 |
| 参与角色 | PMC计划员、生产主管、仓管员 |
| 前置条件 | 已下达生产订单；系统中存在工单用料清单；相关物料库存可查询 |
| 触发条件 | 车间根据生产任务发起领料，或在生产过程中追加辅料 |
| 基本事件流 | 1. PMC下达生产任务。2. 生产主管依据计划和配方发起领料申请。3. 仓管员按批次执行领料出库。4. 生产过程中若因原料属性变化需要追加辅料，则生产主管发起补料申请。5. 仓管员办理补料出库。6. 系统记录领料、补料与对应生产任务之间的关系。 |
| 异常事件流 | 1. 批次库存不足时，系统不允许继续出库。2. 领错料、补料原因不清或未关联生产订单时，系统要求补充说明。3. 若相关物料已过有效期，则系统禁止出库。 |
| 后置条件 | 领料、补料、生产任务和库存台账形成可追溯关系；库存数量同步变更。 |

### 表3-6 用例规约：审批处理与状态写回

| 项目 | 内容 |
| --- | --- |
| 用例名称 | 审批处理与状态写回 |
| 参与角色 | 业务提交人、审批人员、管理层 |
| 前置条件 | 已存在可发起流程的业务单据；流程定义和状态映射已配置 |
| 触发条件 | 业务用户在页面上点击“提交审批”或审批人员在审批中心处理待办任务 |
| 基本事件流 | 1. 业务用户发起审批流程。2. 系统创建流程实例并生成首个待办任务。3. 审批人员在审批中心查看并处理待办。4. 系统根据审批结果推进流程节点。5. 系统读取状态映射规则并写回业务表状态字段。6. 页面刷新后显示新的业务状态。 |
| 异常事件流 | 1. 当前用户无审批权限时，系统拒绝处理任务。2. 状态映射缺失时，流程实例可继续推进，但系统记录“未执行状态写回”。3. 业务单据已失效或被撤回时，不允许继续审批。 |
| 后置条件 | 审批记录被保存；流程实例状态更新；业务表状态与流程节点保持一致或留下异常痕迹。 |
