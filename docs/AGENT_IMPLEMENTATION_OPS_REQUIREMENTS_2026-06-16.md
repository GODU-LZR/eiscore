# EISCore Agent 实施贯标运维系统需求文档

版本：v0.1 初稿
日期：2026-06-16
状态：第二阶段规划初稿
适用项目：EISCore

## 1. 背景

EISCore 面向制造企业，系统能力覆盖业务配置、动态应用、流程、权限、知识库、统计、AI Agent、移动端和未来的智能收单能力。随着客户数量和业务复杂度提升，仅靠人工实施顾问手工调研、配置、编写文档、跟进验收和维护，会导致交付周期长、标准不一致、项目经验难沉淀。

因此，需要搭建一个独立的 Agent 实施贯标运维系统，用于支撑 EISCore 从商务、售前、调研、合同、实施、二次开发、验收文件、验收、售后到维护的全生命周期。

该系统的核心目标不是替代 EISCore 本体业务模块，而是成为 EISCore 的“交付操作系统”和“实施标准化平台”，让系统可以被更快、更标准、更可追溯地配置给客户使用。

## 2. 需求定位

本系统定位为：

```text
面向 EISCore 交付团队的 Agent 实施贯标运维系统。
```

它负责把交付过程中的经验、模板、任务、配置、文档、代码变更、验收、售后问题沉淀成可复用的标准流程，并通过 Agent 自动辅助执行。

一句话描述：

```text
把“怎么卖、怎么调研、怎么配置、怎么开发、怎么验收、怎么维护”变成一套可执行、可复用、可追溯的 Agent 工作系统。
```

### 2.1 阶段定位

本系统属于 EISCore 后续建设的第二阶段主线，不作为当前第一阶段“智能收单与自动入库 Agent”的前置依赖。

阶段划分：

```text
第一阶段：智能收单与自动入库 Agent
第二阶段：Agent 实施贯标运维系统
```

第一阶段优先面向客户现场，解决工厂资料自动识别、自动入库、自动统计的问题。

第二阶段面向 EISCore 交付团队，解决从商务、售前、调研、合同、实施配置、二次开发、验收、售后到维护的标准化交付问题。

第二阶段可以复用第一阶段形成的能力和经验，包括：

1. 智能收单配置模板。
2. 客户单据样本和字段映射经验。
3. 采集端部署经验。
4. 客户现场日志和问题记录。
5. 智能收单验收材料。

## 3. 核心目标

### 3.1 业务目标

1. 缩短 EISCore 项目从商机到上线的周期。
2. 将实施方法论标准化，减少交付质量依赖个人经验。
3. 让售前资料、调研记录、合同范围、实施任务、二开需求、验收文件、售后问题形成连续数据链。
4. 自动生成系统配置建议和配置包。
5. 自动生成项目文档、验收文件、运维记录和客户知识库。
6. 沉淀行业、客户、模块和场景经验，用于后续项目复用。
7. 支持项目范围控制，减少合同外需求失控。

### 3.2 系统目标

1. 建立客户项目全生命周期工作台。
2. 建立标准实施阶段、交付物和检查项。
3. 建立 Agent 可执行任务体系。
4. 建立 EISCore 配置包和部署记录。
5. 建立合同范围、需求、任务、代码、验收、售后之间的追溯关系。
6. 建立实施知识库和项目经验库。
7. 建立售后运维巡检和问题闭环机制。

## 4. 适用业务流程

系统覆盖以下阶段：

```text
商务
-> 售前
-> 调研
-> 合同
-> 项目立项
-> 实施配置
-> 二次开发
-> 测试联调
-> 验收文件
-> 验收
-> 售后
-> 维护
-> 复盘和续约
```

每个阶段都应有：

1. 标准任务。
2. 标准输入。
3. 标准输出。
4. 负责人。
5. Agent 辅助动作。
6. 阶段检查项。
7. 交付物。
8. 风险和问题记录。
9. 与下一阶段的数据交接。

## 5. 与 EISCore 本体的关系

本系统建议独立建设，但要和 EISCore 深度集成。

### 5.1 独立性

独立系统负责：

1. 客户项目管理。
2. 交付流程管理。
3. 实施知识库。
4. Agent 编排。
5. 配置包生成。
6. 售后运维。
7. 文档和验收管理。

### 5.2 集成性

与 EISCore 本体集成：

1. 读取 EISCore 模块、动态应用、字段、流程、权限和配置。
2. 向 EISCore 目标环境写入配置包。
3. 生成客户专属的角色、权限、流程、表单、看板和智能收单配置。
4. 读取客户环境运行状态、日志、版本和问题。
5. 将售后问题转化为配置变更、二开任务或知识库内容。

### 5.3 环境边界

建议至少区分：

```text
模板环境
演示环境
客户测试环境
客户生产环境
```

Agent 可以在模板环境和测试环境自动配置，在生产环境执行配置变更时，需要按企业策略控制。

### 6.1 客户项目空间

1. 客户档案。

4. 调研记录。

8. 配置包。
9. 二开任务。
10. 测试问题。
11. 验收文件。
12. 售后工单。
13. 运维记录。
14. 项目知识库。

### 6.2 实施标准模板

实施标准模板用于定义不同行业、规模和模块组合下的标准交付流程。

模板内容包括：

1. 阶段流程。
2. 标准任务。
3. 标准交付物。
4. 调研问卷。
5. 角色权限模板。
6. 流程模板。
7. 动态应用模板。
8. 字段模板。
9. 报表模板。
10. 验收模板。
11. 培训模板。
12. 运维巡检模板。

### 6.3 配置包

配置包是本系统输出给 EISCore 的核心交付资产。

配置包可以包含：

1. 组织架构。
2. 用户和角色。
3. 权限。
4. 动态应用。
5. 字段。
6. 表格视图。
7. 流程。
8. 状态流转。
9. 编码规则。
10. 基础字典。
11. 主数据模板。
12. 看板和统计。
13. AI 助手配置。
14. 智能收单映射配置。
15. 客户专属 SOP。

配置包应支持版本管理、差异比较、试运行和回滚。

### 6.4 Agent 执行记录

所有 Agent 动作必须有记录：

1. 输入是什么。
2. Agent 判断了什么。
3. 生成了什么。
4. 修改了什么。
5. 调用了哪个工具。
6. 影响了哪个客户环境。
7. 是否需要人工确认。
8. 执行结果是什么。

## 7. 阶段需求

### 7.1 商务阶段

目标：沉淀客户线索、客户背景、业务痛点和初步机会。

功能：

1. 客户档案管理。
2. 联系人管理。
3. 商机阶段管理。
4. 客户行业、规模、组织、系统现状记录。
5. 客户痛点记录。
6. 竞争对手和竞品情况记录。
7. 预算、决策链、时间计划记录。
8. 商务跟进记录。

Agent 能力：

1. 根据沟通记录总结客户痛点。
2. 判断客户适合的 EISCore 模块组合。
3. 生成跟进计划。
4. 提醒缺失关键信息。
5. 生成客户画像。

输出：

1. 客户基础档案。
2. 商机摘要。
3. 初步模块建议。
4. 商务跟进计划。

### 7.2 售前阶段

目标：根据客户情况快速形成演示方案、报价范围和初步解决方案。

功能：

1. 售前需求摘要。
2. 方案模板管理。
3. 演示环境配置建议。
4. 模块组合推荐。
5. 报价项和范围建议。
6. 售前材料版本管理。

Agent 能力：

1. 根据客户行业和痛点生成售前方案。
2. 生成演示脚本。
3. 生成演示数据建议。
4. 生成客户问题回答草稿。
5. 将售前承诺自动标记为后续合同范围候选。

输出：

1. 售前方案。
2. 演示脚本。
3. 模块清单。
4. 报价范围建议。
5. 风险提示。

### 7.3 调研阶段

目标：把客户真实业务流程、岗位、单据、表格、痛点、现有系统和管理规则结构化。

功能：

1. 调研计划。
2. 调研问卷。
3. 访谈记录。
4. 业务流程梳理。
5. 岗位职责梳理。
6. 单据和表格收集。
7. 当前系统现状记录。
8. 数据迁移范围记录。
9. 差异和风险记录。

Agent 能力：

1. 根据行业生成调研提纲。
2. 从访谈记录中抽取流程、岗位、单据和规则。
3. 从客户表格中识别需要的动态应用和字段。
4. 生成业务蓝图草稿。
5. 识别合同外需求或潜在二开需求。
6. 生成系统配置建议。

输出：

1. 调研纪要。
2. 业务流程图。
3. 岗位权限草案。
4. 单据清单。
5. 字段清单。
6. 报表清单。
7. 业务蓝图。
8. 配置包草案。

### 7.4 合同阶段

目标：明确合同范围、交付边界、二开范围、验收标准和付款节点。

功能：

1. 合同资料管理。
2. 合同范围结构化。
3. 模块范围确认。
4. 二开范围确认。
5. 验收标准确认。
6. 付款节点管理。
7. 合同外需求识别。

Agent 能力：

1. 从合同中抽取交付范围。
2. 对比售前承诺和合同范围。
3. 对比调研需求和合同范围。
4. 标记范围风险。
5. 生成合同执行清单。

输出：

1. 合同范围清单。
2. 交付物清单。
3. 验收标准清单。
4. 付款节点清单。
5. 范围风险清单。

### 7.5 项目立项阶段

目标：把合同范围转成项目计划和实施任务。

功能：

1. 项目基础信息。
2. 项目团队。
3. 项目里程碑。
4. WBS 任务。
5. 风险登记。
6. 项目会议纪要。
7. 项目群待办。

Agent 能力：

1. 根据合同范围自动生成实施计划。
2. 根据模块范围生成任务清单。
3. 生成项目启动会材料。
4. 识别缺失前置条件。
5. 生成客户配合事项清单。

输出：

1. 项目计划。
2. 实施任务清单。
3. 项目启动材料。
4. 客户配合清单。

### 7.6 实施配置阶段

目标：快速配置 EISCore，使客户可以尽快试用和上线。

功能：

1. 组织架构配置。
2. 用户和角色配置。
3. 权限配置。
4. 动态应用配置。
5. 字段配置。
6. 表格视图配置。
7. 流程配置。
8. 基础字典配置。
9. 主数据导入。
10. 报表和看板配置。
11. AI 助手配置。
12. 智能收单配置。

Agent 能力：

1. 根据调研资料生成配置方案。
2. 根据客户表格生成动态应用字段。
3. 根据岗位生成权限建议。
4. 根据流程描述生成流程草案。
5. 根据统计需求生成看板和汇总规则建议。
6. 生成配置包。
7. 在测试环境自动执行配置。
8. 生成配置差异报告。

输出：

1. 配置方案。
2. 配置包。
3. 配置执行记录。
4. 配置差异报告。
5. 客户试用环境。

### 7.7 二次开发阶段

目标：管理合同内和合同外二开需求，形成需求、设计、开发、测试、发布闭环。

功能：

1. 二开需求池。
2. 需求范围判定。
3. 优先级管理。
4. 技术方案。
5. 开发任务。
6. 测试用例。
7. 发布记录。
8. 代码和版本关联。

Agent 能力：

1. 判断需求是否属于合同范围。
2. 生成需求说明。
3. 生成技术方案草稿。
4. 拆分开发任务。
5. 生成测试用例。
6. 汇总代码变更和发布说明。
7. 将二开经验沉淀为模板能力。

输出：

1. 二开需求单。
2. 范围判定。
3. 技术方案。
4. 开发任务。
5. 测试报告。
6. 发布说明。

### 7.8 测试联调阶段

目标：验证配置、流程、数据、二开功能和业务场景是否可用。

功能：

1. 测试计划。
2. 测试用例。
3. 场景测试。
4. 缺陷管理。
5. 问题整改。
6. 回归测试。

Agent 能力：

1. 根据业务蓝图生成测试用例。
2. 根据配置包生成测试清单。
3. 根据缺陷描述判断所属模块。
4. 生成回归测试建议。
5. 统计测试通过率和遗留风险。

输出：

1. 测试用例。
2. 测试记录。
3. 缺陷清单。
4. 回归报告。
5. 上线风险评估。

### 7.9 验收文件阶段

目标：自动生成客户验收所需文件，减少手工整理成本。

功能：

1. 验收资料清单。
2. 交付物自动归档。
3. 项目过程资料整理。
4. 系统配置说明。
5. 培训记录。
6. 测试报告。
7. 上线报告。
8. 验收报告。

Agent 能力：

1. 根据项目全过程记录生成验收文件。
2. 根据合同范围核对交付物。
3. 自动整理截图、配置、任务、测试、培训和上线记录。
4. 生成验收问题清单。
5. 生成客户确认文本。

输出：

1. 验收报告。
2. 交付物清单。
3. 系统配置说明书。
4. 用户培训记录。
5. 测试报告。
6. 上线报告。

### 7.10 验收阶段

目标：完成客户确认、问题整改和项目结项。

功能：

1. 验收会议记录。
2. 验收问题清单。
3. 整改任务。
4. 客户确认记录。
5. 验收状态管理。
6. 结项归档。

Agent 能力：

1. 根据验收会议生成问题和整改任务。
2. 判断验收问题是否属于合同范围。
3. 生成整改计划。
4. 生成验收确认材料。
5. 生成项目复盘报告。

输出：

1. 验收确认记录。

3. 项目结项报告。
4. 项目复盘报告。

### 7.11 售后阶段

目标：处理客户上线后的使用问题、优化需求和故障。

功能：

1. 售后工单。
2. 问题分类。
3. 响应时效。
4. 解决方案。
5. 客户反馈。
6. 知识库沉淀。
7. 合同外需求转化。

Agent 能力：

1. 根据客户描述判断问题类型。
2. 关联历史项目、配置、日志和知识库。
3. 给出处理建议。
4. 判断是否为二开需求。
5. 自动生成回复草稿。
6. 将高频问题沉淀为 FAQ 和 SOP。

输出：

1. 售后工单。
2. 处理记录。
3. 解决方案。
4. 知识库条目。
5. 优化需求。

### 7.12 维护阶段

功能：

1. 客户环境清单。
2. 版本管理。
3. 巡检计划。
4. 系统健康检查。
5. 日志监控。
6. 备份检查。
7. 性能检查。
8. 升级计划。
9. 维护报告。

Agent 能力：

1. 自动生成巡检任务。
2. 分析客户端日志、服务端日志和异常。
3. 识别高风险客户环境。
4. 生成维护报告。
5. 生成升级影响评估。
6. 推荐功能优化和增购机会。

输出：

1. 巡检记录。
2. 维护报告。
3. 升级计划。
4. 风险清单。
5. 续约或增购建议。

## 8. Agent 角色设计

系统可内置多个专业 Agent。

### 8.1 商务 Agent

负责客户画像、商机摘要、跟进计划和机会判断。

### 8.2 售前 Agent

负责方案生成、演示脚本、报价范围建议和售前答疑。

### 8.3 调研 Agent

负责调研问卷、访谈纪要、流程梳理、单据识别和业务蓝图生成。

### 8.4 合同 Agent

负责合同范围抽取、范围风险识别、付款节点和验收标准提取。

### 8.5 实施顾问 Agent

负责实施计划、配置建议、客户配合清单和上线准备。

### 8.6 配置 Agent

负责生成 EISCore 配置包，包括动态应用、字段、权限、流程、看板和智能收单配置。

### 8.7 开发 Agent

负责二开需求整理、技术方案草案、任务拆分、测试用例和发布说明。

### 8.8 测试 Agent

负责测试场景、测试用例、缺陷分类、回归建议和上线风险评估。

### 8.9 验收 Agent

负责验收文件生成、交付物核对、验收问题整理和结项报告。

### 8.10 售后 Agent

负责工单分类、解决方案建议、客户回复草稿和知识库沉淀。

### 8.11 运维 Agent

负责巡检、日志分析、异常预警、升级影响评估和维护报告。

## 15. 权限和角色

## 16. 与智能收单系统的关系

智能收单系统是 EISCore 交付给客户后的业务数据自动录入能力。

实施贯标运维系统需要负责：

1. 在调研阶段收集客户表格和单据样本。
2. 在实施阶段生成智能收单映射配置。
3. 在验收阶段生成智能收单测试和验收材料。
4. 在售后阶段分析智能收单识别失败的问题。
5. 在维护阶段优化客户的智能收单模板。

也就是说：

```text
实施贯标运维系统负责把智能收单能力配置好、交付好、维护好。
智能收单系统负责客户现场资料的自动识别和入库。
```

## 18. 验收标准

### 18.1 项目贯标

1. 能按标准阶段管理项目。
2. 每个阶段有标准任务、交付物和检查项。
3. 阶段之间的数据可以自动传递。
4. 能看到项目范围、进度、风险和问题。

### 18.2 Agent 辅助

1. Agent 能根据客户资料生成调研提纲。
2. Agent 能根据调研结果生成业务蓝图草稿。
3. Agent 能生成 EISCore 配置包草案。
4. Agent 能识别合同外需求。
5. Agent 能生成验收文件草稿。

### 18.3 配置交付

1. 能保存配置包版本。
2. 能比较配置差异。
3. 能将配置包部署到测试环境。
4. 能记录配置执行日志。
5. 能追溯配置来源。

### 18.4 售后维护

1. 能记录售后工单。
2. 能关联客户环境、版本和配置。
3. 能沉淀知识库。
4. 能生成维护报告。
5. 能把高频问题转为模板优化。

## 19. 当前待确认问题

1. 该系统是作为 EISCore 的一个独立模块，还是独立仓库和独立部署。
2. 第二阶段启动时，优先做“配置包生成”，还是优先做“项目流程管理”。
3. 是否需要给客户开放部分门户能力，例如查看任务、提交问题、确认验收。
4. 配置包执行到客户生产环境时，是否需要强制人工审批。
5. 二次开发是否要和 GitHub、CI、发布流程打通。
6. 文档生成优先支持 Word，还是先支持 Markdown/PDF。
7. 运维巡检需要接入哪些指标，例如日志、数据库、服务状态、客户端状态。
8. 行业模板第一批选择哪些行业和模块组合。

## 20. 一句话总结

本系统要建设的是 EISCore 的“交付操作系统”：用 Agent 把商务、售前、调研、合同、实施配置、二次开发、验收、售后和维护串成一条标准化、自动化、可追溯的交付链路，最终目标是让 EISCore 可以更快被配置、更稳被交付、更持续地被维护。
