# EISCore 证据链说明

本文档用于记录 EISCore 项目的公开证据链材料范围。证据链材料用于说明代码形成时间、持续开发过程、授权边界、文档来源和作者身份，不构成任何权利转让、买断、职务作品确认、委托开发成果归属确认、独占授权或权利放弃。

## 证据材料

1. GitHub commit 记录：用于证明 EISCore 代码形成时间、持续开发过程、功能演进和作者维护轨迹。
2. Git tag / release：用于固定公开发布时间和公开版本边界，建议按 `v0.1-personal-public`、`v0.2-agpl-declared`、`v0.3-thesis-evidence` 等版本节点留痕。
3. README、LICENSE、NOTICE：用于证明项目授权边界、版权归属声明和公开发布说明。
4. 源码 SPDX 头：用于证明具体源码文件适用的许可证和版权归属声明。
5. COPYRIGHT.md：用于记录项目名称、版权人、首次公开年份、许可证和第三方依赖边界。
6. COMMERCIAL-LICENSE.md：用于说明 AGPL-3.0-or-later 开源授权与单独书面商业授权并行，防止开源发布被解释为商业买断或专有权属转让。
7. 微信/邮件确认：用于证明与渊途之间不存在 EISCore 权属转让、买断、职务作品确认、委托开发成果归属确认或独占授权关系。
8. 项目文档、论文、开题报告：用于证明 EISCore 的设计来源、系统架构、核心实现思路、作者身份和毕业设计时间线。
9. 哈希存证/可信时间戳/公证：用于增强公开材料的内容固定性、时间固定性和可核验性。
