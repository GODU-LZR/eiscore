# 设计总说明与文献引用合规方案

题目：面向湛江中小制造企业的企业信息化系统核心设计与实现——以广东南派食品有限公司为例
整理日期：2026-03-14
用途：用于毕业论文初稿写作，包括设计总说明初稿、真实参考文献获取方法、引用核验流程，以及合规降低重复率和 AI 风险的写作建议。

## 一、设计总说明初稿

当前，制造业数字化转型已成为提升企业经营效率和市场竞争力的重要路径，但对于湛江地区大量中小制造企业而言，问题往往不在于“是否已经上系统”，而在于关键业务记录能否在同一条数据链路中稳定流转。以广东南派食品有限公司为例，库存台账、领料记录、检验附件和审批状态一旦分散在纸单、聊天消息和独立台账中，后续再追查某批原料去了哪个仓位、是否发生过补料、最终进入哪张出库单时，管理效率就会明显下降。与此同时，企业还长期面临预算有限、系统定制成本偏高、专职运维人员不足以及现场规则经常变化等现实约束。传统大型 ERP 系统虽然功能完整，但部署周期长、实施成本高；部分轻量工具虽然上线较快，却很难在跨模块协同、字段扩展和流程联动上持续适应业务变化。因此，围绕中小制造企业构建一种轻量、可扩展、便于运维且能够适应现场变化的企业信息化底座，具有较强的现实意义。

针对上述问题，本文设计并实现了 EISCore 企业信息化系统核心平台。系统以数据库中心架构为基本思路，以 PostgreSQL 作为核心数据与业务逻辑承载平台，结合 PostgREST 自动映射接口，弱化传统后端服务层在接口开发中的重复劳动；前端采用 Vue 3 技术栈，并通过 qiankun 构建微前端基座，将人事管理、物料管理、应用中心和移动端等业务模块进行拆分与集成。在业务实现上，系统围绕 HMS 与 MMS 两个核心域展开，已实现员工档案、组织结构、用户权限、考勤管理、物料主数据、仓库管理、库存台账、入库、出库、盘点等主要功能，并进一步集成 BPMN 流程设计与运行能力，用于支撑审批流转和状态变化。与此同时，本文在系统建设过程中进一步引入轻量本体语义增强机制，将表级语义、列级语义、表间关系与流程状态纳入统一描述框架，使系统在动态应用配置、流程协同和语义治理方面具有更好的可扩展性。整体来看，本课题不仅关注企业信息系统“能否实现”，也关注系统在业务表达、结构扩展和后续智能化演进方面的可持续性。

本文围绕 EISCore 系统的设计与实现过程展开论述。第一章主要介绍课题的研究背景、研究意义以及国内外相关研究现状；第二章介绍系统所使用的关键技术与开发环境；第三章从系统目标、用户角色和业务场景出发，对系统需求进行分析；第四章给出系统总体架构、模块划分和关键设计思路；第五章重点描述数据库结构设计、核心数据模型以及本体语义增强机制设计；第六章对人事管理、物料管理、流程协同和动态应用配置等核心模块进行详细设计说明；第七章展示系统的实现过程与运行效果；第八章结合测试环境、功能测试和结果分析对系统进行验证；最后对全文进行总结，并对系统目前存在的不足和未来改进方向进行展望。

## 二、真实参考文献如何获取，避免“AI 造假引用”

### 1. 基本原则

毕业论文中的参考文献不能依赖大模型“直接生成”。正确做法是：

1. 先到真实数据库中检索到文献。
2. 再从数据库页面或文献管理工具中导出题录。
3. 最后人工核对题名、作者、期刊、年份、卷期、页码和链接是否一致。

换句话说，AI 只能帮你确定检索关键词、筛选主题和梳理研究脉络，不能代替你生成最终参考文献清单。

### 2. 中文文献建议检索路径

中文文献优先从以下渠道检索：

1. 中国知网 CNKI
2. 万方
3. 维普
4. 学校图书馆数据库入口

对于你这个题目，建议优先搜以下主题词：

1. 中小制造企业 信息化
2. 企业信息化系统 设计与实现
3. 微前端 企业系统
4. PostgREST 数据库中心架构
5. BPMN 工作流 企业应用
6. 本体 语义建模 企业系统
7. 低代码平台 制造业
8. PostgreSQL 行级安全 RLS

### 3. 英文文献建议检索路径

英文文献建议优先从以下数据库获取：

1. Google Scholar
2. Web of Science
3. ACM Digital Library
4. IEEE Xplore
5. Crossref Metadata Search

其中：

1. Google Scholar 适合做广泛检索和追踪被引。
2. Web of Science 适合筛高质量英文论文并导出 BibTeX。
3. ACM DL 和 IEEE Xplore 适合找软件工程、系统架构、工作流、前端工程相关论文。
4. Crossref 适合做 DOI 和元数据核验。

## 三、真实参考文献的操作流程

### 1. 中文文献操作流程

联网核验到的高校图书馆指南显示，CNKI 支持直接导出 GB/T 7714 格式题录。较稳妥的操作流程是：

1. 在 CNKI 检索目标主题。
2. 勾选符合题目的文献。
3. 点击“导出/参考文献”。
4. 在“文献管理中心-文献输出”页面导出 GB/T 格式题录。
5. 导出后不要直接无脑粘贴，要人工复核。

人工复核时重点检查：

1. 作者姓名是否完整。
2. 题名是否与原文一致。
3. 期刊名、年份、卷号、期号、页码是否缺失。
4. 是否混入 DOI 网址、数据库链接、访问路径等学校不需要的内容。

### 2. 英文文献操作流程

英文文献建议使用“数据库导出 + 人工核验”的方式：

1. 在 Google Scholar 搜索主题，先确定候选论文。
2. 点击 Scholar 检索结果下方的 “Cite”。
3. 导出 BibTeX、EndNote、RefMan 或 RefWorks。
4. 若文章来自 ACM DL，可在论文页面直接导出 BibTeX 或 EndNote。
5. 若文章来自 Web of Science，可导出 BibTeX。
6. 将导出的条目导入 Zotero、NoteExpress 或 EndNote。
7. 最后再统一按 GB/T 7714 或学校模板输出。

### 3. 推荐使用文献管理工具

如果你后面要频繁改格式，建议不要手工维护几十条参考文献，推荐至少选一个：

1. Zotero
2. NoteExpress
3. EndNote

优势在于：

1. 可以保存原始题录来源。
2. 可以统一改格式。
3. 可以插入正文引用。
4. 后续导师要求增删文献时修改成本低。

## 四、如何防止参考文献“看起来像真的，其实是假的”

### 1. 每条文献至少做一次“四项核验”

每条参考文献至少核验以下四项：

1. 是否能在数据库中重新检索到原文。
2. 作者、题名、年份是否完全对应。
3. 期刊卷期页码是否存在。
4. DOI 或出版链接是否可追溯。

### 2. 建议建立“文献证据表”

你可以自己建一个 Excel 或 Markdown 表，字段建议如下：

1. 编号
2. 中文题名 / 英文题名
3. 作者
4. 来源数据库
5. 发表年份
6. 卷期页码
7. DOI 或链接
8. 是否已读
9. 准备引用到哪一章

这样后面导师抽查时，你能快速证明这些文献是真实存在的。

### 3. 不要直接使用 AI 生成整段参考文献

这是最容易出问题的地方。大模型常见问题包括：

1. 把真实作者和虚构题名拼在一起。
2. 生成不存在的卷期页码。
3. 给出错误 DOI 或死链接。
4. 把会议论文写成期刊论文。

因此，参考文献清单必须以数据库导出结果为准，AI 只能做辅助筛选和整理。

## 五、如何合规降低查重率

这里的原则不是“规避系统”，而是“用规范写作自然降低重复率”。从公开写作规范和 Turnitin 的说明来看，查重报告反映的是文本匹配情况，不等于直接认定抄袭；即使是已正确引用的内容，也可能产生匹配，因此关键在于写法是否规范、引用是否清楚。

建议这样做：

### 1. 不要大段照搬开题报告和中期报告

开题报告和中期报告可以作为你论文的基础材料，但不能整段原封不动搬到初稿里。正确做法是：

1. 保留核心观点。
2. 按论文章节需要重新组织语言。
3. 增加新的分析和过渡。
4. 把阶段性口吻改成正式论文口吻。

### 2. 尽量写“分析”，少写“套话”

重复率高往往不是因为观点重复，而是因为大量使用通用化表述。比如：

1. 不要反复写“随着信息技术的不断发展”这类空话。
2. 多写你这个项目的具体场景、对象、模块和问题。
3. 多结合南派食品、多批次原料、流程协同、权限治理、本体语义等具体内容。

### 3. 对文献做“消化后转述”

正确的转述方式不是简单替换同义词，而是：

1. 先读懂原文。
2. 提炼观点。
3. 用自己的逻辑把观点写进段落。
4. 在句末或段末标注文献。

如果只是改几个词，仍然可能构成高重复。

### 4. 图表自己画，文字自己解释

需求图、架构图、E-R 图、流程图、时序图如果由你自己基于项目实际重画，再配上你自己的解释性文字，通常比单纯堆技术描述更不容易出现高重复。

## 六、如何合规降低“AI 味”和 AI 风险

这里不建议也不提供“规避 AI 检测”的技巧，因为那本质上属于规避审查。更稳妥的方式是让你的论文本身就是你自己的写作成果。联网查到的高校 AI 学术诚信指南也强调，当前 AI 检测工具并不可靠，很多学校更看重是否存在假文献、假链接、与本人写作风格差异过大等问题。

建议这样做：

### 1. AI 只用于辅助，不用于直接交稿

你可以让 AI 帮你：

1. 列提纲
2. 解释概念
3. 帮你梳理句子是否通顺
4. 提醒你哪些地方缺图缺表

但正文一定要自己改写、自己补细节、自己核对事实。

### 2. 保留你的写作痕迹

建议保留：

1. 每一版论文修改记录
2. 你的草稿文件
3. 文献检索记录
4. 图表草图
5. 导师意见修改痕迹

这些都是证明论文是你自己逐步写出来的有效材料。

### 3. 多写项目中的真实细节

越是真实项目细节，越不像模板文：

1. 你为什么引入本体语义，而不是一开始就写进开题报告。
2. 你在项目推进中遇到过哪些模块协同问题。
3. 为什么数据库中心架构适合这个课题。
4. 为什么当前系统完成度约 80%，哪些部分已完成，哪些还在完善。

### 4. 避免“统一腔调”的段落

AI 写作常见问题是每段都很圆滑、很平均、很像总结报告。你在定稿时要主动处理：

1. 有些地方写得更具体。
2. 有些地方写得更像分析，不只是结论。
3. 删掉重复表达。
4. 避免一段里连续堆三个“首先、其次、最后”。

## 七、最稳妥的实际执行方案

结合你现在的时间节点，建议按下面流程操作：

1. 先确定论文目录和各章小节。
2. 按每章内容反推需要的中文文献和英文文献主题。
3. 先从 CNKI 和 Google Scholar 检索候选文献。
4. 对核心英文论文再去 Web of Science、ACM DL、IEEE Xplore 或 Crossref 做元数据核验。
5. 把所有题录导入文献管理工具。
6. 写论文时边写边引，不要最后集中补参考文献。
7. 每完成一章，自己做一次“事实核对 + 引文核对 + 语言重写”。

## 八、对你当前论文最直接的建议

1. 设计总说明可以先用本文第一部分作为初稿。
2. 第一章绪论中的国内外研究现状，不要靠 AI 直接生成，先把真实论文找出来再写。
3. “本体论”部分建议在论文中统一写成“轻量本体语义增强机制”，更稳妥，也更容易和系统实现对应。
4. 如果学校当前模板仍按 GB/T 7714-2015 执行，就先按 2015 版整理；虽然 2026 年已有 GB/T 7714-2025 发布信息，但其实施时间在 2026-07-01 之后，毕业论文应优先服从学校当年模板和学院要求。

## 九、联网核验依据

以下内容用于支撑本文中的操作建议：

1. Turnitin 官方说明指出，相似度报告反映的是文本匹配，不直接等于抄袭认定，是否构成抄袭仍需学术判断。
2. Vanderbilt University 的 AI 学术诚信指南指出，目前没有万无一失的 AI 检测方法，且不能仅凭 AI 检测分数认定违规。
3. 清华大学图书馆写作规范页面提到，参考文献整理可依据 GB/T 7714，并可借助 EndNote、NoteExpress 等工具完成管理与插入。
4. 中国人民大学图书馆的题录导出指南明确写到，可从 CNKI 导出 GB/T 格式题录，从 Web of Science 导出 BibTeX。
5. Google Scholar 官方帮助说明支持通过 “Cite” 功能导出 BibTeX、EndNote、RefMan、RefWorks。
6. ACM Digital Library 官方帮助页面说明支持导出 BibTeX、EndNote、ACM Ref 格式的引文。

## 十、后续可以继续做的事情

如果按这个方案继续推进，下一步最合适的是：

1. 直接开始检索并筛出与你课题最相关的 10 篇中文文献和 2 篇英文文献。
2. 我根据这些真实文献帮你写第一章“绪论”的文献综述部分。
3. 再继续生成英文版设计总说明，并同步输出 Word 文档。
