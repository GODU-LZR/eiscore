## 1. 本轮识别到的老师批注重点

根据老师批注版 Word 和你补充上传的截图，本轮需要重点处理的问题主要有：

1. 说明书属于设计类文稿，不应大量使用“本文”“本论文”“本课题”等论文式口径，应统一改为“本设计”“本系统”。
2. 第三章的总体功能结构图应放在各模块用例图之前，用于先总述系统功能范围，再进入模块化需求分析。
3. 总体功能结构图存在“线条杂乱、字太小”的问题，需要重绘成更适合 Word 横向插入的版本。
4. 用例规约表不能简单连续堆叠，每张表前都要有说明和正文引用。
5. 活动图应跟在对应模块的用例规约之后，而不是集中放在后面。
6. 第四章应明确写成“系统概要设计”，相关小节标题也要统一成概要设计口径。
7. 图4-1附近的表述应强调“架构”，不宜混用“结构”。
8. 系统总体架构图中的连线应尽量规整，避免弯曲过多。
9. “功能模块划分”若与第三章需求分析重复过多，应避免机械重复，转为关键设计思路和组织方式说明。
10. 第六章详细设计最需要加强，不应停留在表结构和函数名罗列，而应交代清楚前端输入、接口调用、数据库函数、RLS/权限控制、状态写回等实际实现链路。
11. 库存入库与出库部分不应重复需求分析，要突出详细处理过程。
12. 轻量化语义模块不能写成通用模块说明，而要结合本体关系工作台、应用中心、FlashBuilder 和 Agent Runtime 的实际实现来写。

## 2. 本轮已经落实的修改

### 2.1 全文口径统一

已将主稿中的“本文”“本论文”“本课题”等表述统一替换为“本设计”或“本系统”，使文稿更符合设计说明书的写法。

### 2.2 第三章需求分析重组

1. 将总体功能结构图放在各模块用例图之前。
2. 对图3-6前后的文字进行了重写，明确其作用是概括主实现内容与支撑能力，而不是重复目录。
3. 对表3-3至表3-6的用例规约表增加了单独的说明段，避免多张表直接堆叠。
4. 将采购入库与来料检验活动图、生产领料与补料活动图放在对应规约之后。

### 2.3 第四章概要设计口径修正

1. 章节标题统一为“系统概要设计”。
2. 小节标题统一为“系统概要设计目标”等概要设计口径。
3. 图4-1前后的描述统一改为“架构”，避免“结构/架构”混用。
4. 去掉与第三章重复较重的“功能模块划分”展开方式，当前重点落在关键设计思路和组织关系说明。

### 2.4 第六章详细设计强化

1. 用户与权限控制部分，改成“前端状态识别 -> PostgREST 传递 JWT -> 数据库函数/RLS裁决”的链路说明。
2. 库存部分，重点改为“页面参数如何进入 `scm.stock_in` / `scm.stock_out`，并在事务内同时更新批次与流水”。
3. 流程部分，重点改为“流程定义 -> 实例启动 -> 任务判断 -> 状态写回”的链路说明。
4. 轻量化语义部分，明确写出与本体关系工作台、应用中心、FlashBuilder、Agent Runtime 的关系，不再停留在通用概念说明。
